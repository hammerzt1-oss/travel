## 技术规范

### 技术栈（已定死）
- **框架**：Next.js 14 + TypeScript
- **样式**：Tailwind CSS
- **状态管理**：React Context（简单场景）
- **部署**：Vercel（免费）

### API接口规范

**推荐列表API：**
```
GET /api/recommendations?type=week|month|popular
```

**响应数据结构：**
```typescript
{
  code: 200,
  data: {
    list: [
      {
        id: number,
        name: string,
        tag: string,
        budget_range: string,
        primary_reason: string, // 新增：直接用于CTA上方文案
        transport: string,
        weather: string,
        suitable_days: string,
        trust_signals: {
          student_count: number,
          click_count_7d: number,
          is_popular: boolean
        },
        cover_image: string,
        cta_text: string
      }
    ]
  }
}
```

### 信任信号显示规则
- ❌ 一张卡里3个信任信号 = 信息噪音
- ✅ 每张卡最多显示1-2个
- 普通卡：显示"已被XXX名学生选择"
- 主推卡：显示"🔥 最近7天点击最多"

### OTA链接处理
- ❌ **不要在前端拼返佣参数**
- ✅ **所有pid/utm/source只在后端**
- ✅ 前端只拿`cta_links.hotel`等完整链接
- ✅ 直接使用后端返回的完整链接
