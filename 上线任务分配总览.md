## 👥 角色任务分配

### 后端工程师（最高优先级）

**任务：** 部署后端到Railway/Render

**预计时间：** 30-40分钟

**关键步骤：**
1. 选择部署平台（Railway或Render）
2. 部署后端服务
3. 配置环境变量（OTA_PID=284116645）
4. 验证部署（/health、API、OTA链接）
5. **提供后端URL给前端工程师**

**详细说明：** 见 `后端-上线任务.md`

**完成标准：**
- [ ] 部署成功
- [ ] /health返回200
- [ ] API能正常返回数据
- [ ] OTA链接包含正确PID

---

### 前端工程师（高优先级）

**任务：** 部署前端到Vercel

**预计时间：** 30分钟

**前提条件：** 等待后端工程师完成部署并获取后端URL

**关键步骤：**
1. 连接GitHub仓库到Vercel
2. 配置环境变量（NEXT_PUBLIC_API_URL）
3. 部署前端
4. 验证部署（访问、功能、API、移动端）

**详细说明：** 见 `前端-上线任务.md`

**完成标准：**
- [ ] 部署成功
- [ ] 网站能正常访问
- [ ] 推荐列表正常显示
- [ ] API调用正常

---

### 运营（中优先级）

**任务：** 准备数据观察 + 上线后验证

**预计时间：** 上线后30分钟（验证）+ 持续观察

**关键步骤：**
1. 准备数据观察工具（Google Analytics或日志）
2. 配合Step 3验证（功能、数据、文案）
3. **执行Step 4验证（最关键：联盟后台点击记录）**
4. 执行度假CTA返佣验证（上线后第1天）
5. 开始数据观察（只看3个指标）

**详细说明：** 见 `运营-上线任务.md`

**完成标准：**
- [ ] Step 4验证完成（看到点击记录）
- [ ] 度假CTA返佣验证完成
- [ ] 数据观察工具准备完成
