# 上线后数据观察计划

## 🎯 观察目标

**核心目标：验证"能点出去"的模型是否跑通**

**只看3个指标，不看其他：**
1. 首页推荐卡CTR（点击率）
2. CTA点击次数
3. 是否真实跳到OTA（哪怕只有1次）

---

## 📊 必须监控的3个指标

### 指标1：首页推荐卡CTR（点击率）

**定义：**
```
CTR = 点击推荐卡片的次数 / 访问首页的次数 × 100%
```

**监控方法：**
- Google Analytics事件追踪
- 或简单日志记录

**目标值：**
- **最低目标：** ≥5%
- **理想目标：** ≥10%

**如何追踪：**
```javascript
// 前端：记录推荐卡片点击
const handleCardClick = (destinationId) => {
  // Google Analytics
  gtag('event', 'recommendation_click', {
    destination_id: destinationId
  })
  
  // 或简单日志
  console.log('推荐卡片点击:', {
    destination_id: destinationId,
    timestamp: new Date().toISOString()
  })
}
```

**观察频率：**
- 每天观察一次
- 记录每日CTR
- 观察趋势

---

### 指标2：CTA点击次数

**定义：**
- 每个CTA按钮的点击次数
- 哪个CTA点击最多
- 点击时间分布

**监控方法：**
- Google Analytics事件追踪
- 或后端日志记录

**目标值：**
- **最低目标：** 每天≥10次点击
- **理想目标：** 每天≥50次点击

**如何追踪：**
```javascript
// 前端：记录CTA点击
const handleCTAClick = (type, destinationId) => {
  // Google Analytics
  gtag('event', 'cta_click', {
    cta_type: type, // hotel, transport, package
    destination_id: destinationId
  })
  
  // 或简单日志
  console.log('CTA点击:', {
    type,
    destination_id: destinationId,
    timestamp: new Date().toISOString()
  })
}
```

**观察内容：**
- 哪个CTA点击最多（酒店/交通/套餐）
- 哪个目的地CTA点击最多
- 点击时间分布（什么时候点击多）

**观察频率：**
- 每天观察一次
- 记录每日点击次数
- 分析点击模式

---

### 指标3：是否真实跳到OTA（哪怕只有1次）

**定义：**
- OTA链接是否正常跳转
- 链接是否包含正确PID（284116645）
- 是否有用户完成跳转

**监控方法：**
- 检查服务器日志
- 检查OTA后台数据（如果有）
- 手动测试验证

**目标值：**
- **最低目标：** 至少1次成功跳转
- **理想目标：** 每天≥5次跳转

**如何追踪：**
```javascript
// 后端：记录OTA链接生成
console.log('OTA链接生成:', {
  destination: destination.name,
  cta_type: type,
  link: otaLink,
  timestamp: new Date().toISOString()
})
```

**验证方法：**
1. **手动测试**
   - 每天手动点击一次CTA
   - 验证链接是否正常
   - 验证PID是否正确

2. **日志检查**
   - 检查服务器日志
   - 查找OTA链接生成记录
   - 统计跳转次数

3. **OTA后台（如果有）**
   - 登录携程联盟后台
   - 查看点击数据
   - 验证PID是否正确

**观察频率：**
- 每天验证一次
- 记录是否成功跳转
- 记录跳转次数

---

## ❌ 不要监控的指标

### 不要看转化率
- **原因：** 转化率是OTA的事，不是我们的核心指标
- **影响：** 会分散注意力，影响核心目标

### 不要看成交金额
- **原因：** 成交金额是OTA的事，我们只关心点击
- **影响：** 会让我们追求错误的目标

### 不要看用户停留时间
- **原因：** 不是核心指标，不影响"能点出去"
- **影响：** 会让我们优化错误的方向

## 🔍 数据分析方法

### CTR分析
- **如果CTR < 5%：**
  - 检查推荐卡片是否吸引人
  - 检查信任信号是否有效
  - 优化卡片设计

- **如果CTR ≥ 5%：**
  - 说明推荐有效
  - 继续优化提升
  - 分析哪个推荐类型最有效

### CTA点击分析
- **如果点击少：**
  - 检查CTA按钮是否醒目
  - 检查CTA文案是否"像钱"
  - 优化CTA位置和样式

- **如果点击多：**
  - 分析哪个CTA最有效
  - 分析哪个目的地最受欢迎
  - 优化热门目的地的推荐

### OTA跳转分析
- **如果没有跳转：**
  - 检查链接是否正确
  - 检查PID是否正确
  - 手动测试验证

- **如果有跳转：**
  - 说明模型跑通
  - 继续优化提升
  - 分析跳转模式
