# 前端工程师 - 上线任务

## 🎯 你的任务：部署前端到Vercel

**预计时间：** 30分钟
**优先级：** 高（在后端部署完成后执行）

---

## 📋 执行步骤

### Step 1：准备部署（5分钟）

**前提条件：**
- ✅ 等待后端工程师完成部署并获取后端URL
- ✅ 后端 `/health` 返回200
- ✅ 后端API能正常返回数据

**准备事项：**
- [ ] 确认代码已提交到GitHub
- [ ] 确认 `frontend/.env.local` 已创建（本地开发用）
- [ ] 准备生产环境API地址（从后端工程师获取）

---

### Step 2：部署到Vercel（15分钟）

**步骤：**

1. **访问Vercel**
   - 打开 https://vercel.com
   - 使用GitHub账号登录

2. **导入项目**
   - 点击 "Add New..." → "Project"
   - 选择项目仓库
   - 点击 "Import"

3. **配置项目**
   - **Framework Preset：** Next.js（自动识别）
   - **Root Directory：** `frontend`
   - **Build Command：** `npm run build`（自动）
   - **Output Directory：** `.next`（自动）
   - **Install Command：** `npm install`（自动）

4. **配置环境变量**
   - 在 "Environment Variables" 中添加：
     - **Name：** `NEXT_PUBLIC_API_URL`
     - **Value：** `https://你的后端URL`（从后端工程师获取）
     - **Environment：** Production, Preview, Development（全选）

5. **部署**
   - 点击 "Deploy"
   - 等待部署完成（约2-3分钟）

---

### Step 3：验证部署（10分钟）

**验证项：**

1. **访问验证**
   - [ ] 访问部署后的URL（如：https://xxx.vercel.app）
   - [ ] 网站能正常打开
   - [ ] 没有错误页面

2. **功能验证**
   - [ ] 首页推荐列表正常显示
   - [ ] 推荐卡片信息完整
   - [ ] 占位图正常显示
   - [ ] 点击卡片能跳转到详情页

3. **API验证**
   - [ ] 打开浏览器开发者工具（F12）
   - [ ] 查看Network标签
   - [ ] 确认API请求正常（状态码200）
   - [ ] 确认请求地址是生产环境后端URL

4. **移动端验证**
   - [ ] 切换到移动端视图
   - [ ] 布局正常
   - [ ] 点击区域足够大

---

## ✅ 完成标准

**必须全部通过：**
- [ ] 部署成功
- [ ] 网站能正常访问
- [ ] 推荐列表正常显示
- [ ] API调用正常
- [ ] 移动端体验良好

---

## 📞 如果遇到问题

**部署失败：**
- 检查构建日志
- 检查环境变量配置
- 检查代码是否有错误

**API调用失败：**
- 检查 `NEXT_PUBLIC_API_URL` 是否正确
- 检查后端服务是否正常运行
- 检查CORS配置

**直接说：**
- "前端已部署，生产URL是XXX"
- "前端部署失败，错误是XXX"
- "前端已部署，但API调用失败"

---

## ⚠️ 重要提醒

1. **必须等后端部署完成**
   - 先获取后端URL
   - 再配置前端环境变量
   - 最后部署前端

2. **环境变量必须配置**
   - `NEXT_PUBLIC_API_URL` 必须指向生产环境后端
   - 不要使用 `http://localhost:3001`

3. **部署后立即验证**
   - 不要部署完就走
   - 必须验证功能正常
   - 发现问题立即反馈
